# Contributing

We welcome contributions to improve deployment flexibility and reliability.

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your feature or fix
3. **Make your changes**
4. **Test** locally (`docker compose config`)
5. **Open a Pull Request** against `main`

## Issues

- **Bugs**: Please use the Bug Report template.
- **Features**: Please use the Feature Request template.

## Guidelines

- Keep configurations secure and simple.
- Document any new environment variables.
- Ensure all YAML files are valid.
